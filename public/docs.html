Developer docs

Build on the ghost EMF feed.

The API is public, read-only, edge-cached, and designed for polling. Use the current endpoint for live reads, history for graphs and replay, the entropy endpoint for derived metrics, timeline metadata for annotated browsing, and D1-backed snapshots for export and reproducible random numbers.

Caching

Cloudflare in front of Home Assistant

The current endpoint is cached briefly at the edge to protect the upstream sensor source while staying friendly to polling workloads. History and entropy responses use separate bounded TTLs.

  • Current: short TTL for near-live polling
  • History: longer TTL for charts
  • Entropy: cached derived view over bounded windows
  • Snapshot exports come from D1 rows written every minute by cron
  • Cache API is edge-local, so global HIT behavior can vary by data center

Rate limits

Public, but not abusable

The API is read-only and rate-limited per IP. Every response includes a request ID and rate-limit headers so clients can back off cleanly.

  • X-Request-ID for tracing
  • X-RateLimit-Limit for total budget
  • X-RateLimit-Remaining for remaining calls
  • CORS is explicit for browser-based polling from other domains

Examples

Use it from curl, JavaScript, Python, or Google Sheets

curl https://ghost.megabyte.space/api/v1/ghost-emf/current
const history = await fetch(
  "https://ghost.megabyte.space/api/v1/ghost-emf/history?start=2026-04-03T02:47:58.394Z&end=2026-04-23T23:00:00.000Z&targetPoints=960"
).then((r) => r.json())
import requests

timeline = requests.get(
    "https://ghost.megabyte.space/api/v1/ghost-emf/timeline",
    timeout=10,
).json()
=IMPORTDATA("https://ghost.megabyte.space/api/v1/ghost-emf/export?start=2026-04-03T02:47:58.394Z&end=2026-04-23T23:00:00.000Z&format=csv")

For a rolling window, you can still use minutes. For a full browser experience, prefer explicit start and end timestamps plus targetPoints.
